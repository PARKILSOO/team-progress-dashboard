# 팀 진행상황 대시보드 - 제품 요구사항 문서 (PRD)

**상태**: ready-for-agent  
**작성일**: 2026-06-24  
**개발 기간**: 3주 (21일)

---

## 문제 정의

10명 이상의 팀원을 관리하는 팀장들이 현재 진행상황 파악에 어려움을 겪고 있습니다:

- **수동 추적**: 진행상황을 엑셀로 관리하며 지속적인 수동 업데이트 필요
- **반응형 파악**: 팀장이 각 팀원에게 일일이 물어봐야만 상황 파악 가능
- **정보 오래됨**: 실시간 업데이트가 불가능해 정확도 저하
- **행정 업무 증가**: 상태 업데이트에 전략적 업무 시간 소모
- **정보 종합 불가**: 원본 진행 데이터를 분석하거나 병목 현상 파악 어려움

결과적으로 팀장은 신속한 의사결정을 내릴 수 없고, 팀원들은 반복적인 상태 보고에서 오는 마찰을 경험합니다.

---

## 솔루션

팀장이 다음을 할 수 있도록 하는 실시간 진행상황 대시보드 구축:

1. **한눈에 팀 전체 진행상황 파악** — 10명 팀원의 모든 작업 상태를 하나의 통합 화면에서 확인
2. **자동 업데이트 받기** — 팀원이 진행상황을 입력하면 1초 내에 대시보드에 반영
3. **빠른 정보 해석** — 색상 코드 (진행중=파란색, 완료=초록색, 대기=회색)로 빠른 스캔 가능
4. **팀 병목 이해** — AI 기반 요약이 자동으로 진행 데이터를 종합해 주요 블로커, 완료율, 트렌드 파악

팀원들은 간단하고 마찰 없는 진행상황 입력 경험을 제공받습니다.

## 사용자 스토리

### 팀장 (주요 사용자)

1. 팀장으로서, 나는 모든 팀원(10명)의 모든 작업 진행상황을 하나의 대시보드에서 보고 싶습니다. 그래야 팀 전체 상태를 한눈에 파악할 수 있습니다.

2. 팀장으로서, 나는 팀원의 진행상황 입력 후 1초 이내에 대시보드에 업데이트되는 것을 보고 싶습니다. 그래야 항상 최신 상태를 볼 수 있습니다.

3. 팀장으로서, 나는 진행상황 상태를 색상으로 구분하고 싶습니다 (진행중=파란색, 완료=초록색, 대기=회색). 그래야 빠르게 스캔하고 문제 영역을 찾을 수 있습니다.

4. 팀장으로서, 나는 각 팀원의 작업별 마지막 업데이트 시간을 보고 싶습니다. 그래야 정보가 얼마나 최신인지 알 수 있습니다.

5. 팀장으로서, 나는 모든 팀원의 진행상황 입력을 AI가 종합 분석한 팀 현황 요약을 보고 싶습니다. 그래야 개별 항목을 읽지 않고도 블로커와 전체 완료율을 이해할 수 있습니다.

6. 팀장으로서, 나는 AI 요약이 진행상황 입력에서 언급된 주요 블로커를 찾아내고 표면화하기를 바랍니다. 그래야 차단 해제 작업을 우선순위로 설정할 수 있습니다.

7. 팀장으로서, 나는 AI 요약이 한 문장의 스냅샷을 제공하기를 바랍니다 (예: "5명 진행중, 3명 완료, 2명 대기 | 주요 블로커: 데이터 검증, API 성능"). 그래야 이해관계자들에게 빠르게 보고할 수 있습니다.

8. 팀장으로서, 나는 지난 7일간의 진행상황 히스토리를 보고 싶습니다. 그래야 트렌드를 이해하고 지연 패턴을 감지할 수 있습니다.

9. 팀장으로서, 나는 팀원 명단을 관리하고 싶습니다 (팀원 추가/제거). 그래야 팀 구성이 바뀔 때 대시보드를 최신으로 유지할 수 있습니다.

10. 팀장으로서, 나는 작업 목록을 관리하고 싶습니다 (새 작업 추가, 설명 수정). 그래야 대시보드가 현재 업무를 반영하게 합니다.

11. 팀장으로서, 나는 어느 브라우저에서나 대시보드에 접근하고 싶습니다. 그래야 책상에서, 회의 중, 이동 중에도 진행상황을 확인할 수 있습니다.

### 팀원 (보조 사용자)

12. 팀원으로서, 나는 최소한의 마찰로 빠르게 작업 진행상황을 업데이트하고 싶습니다. 그래야 업무를 방해하지 않으면서 상태를 보고할 수 있습니다.

13. 팀원으로서, 나는 미리 정의된 상태 옵션 중 선택하고 싶습니다 (진행중, 완료, 대기). 그래야 상태 입력이 빠르고 일관성 있게 진행됩니다.

14. 팀원으로서, 나는 진행상황을 간단한 텍스트로 설명하고 싶습니다 (예: "API 인증 모듈 작업 중, 토큰 갱신에서 차단 문제 발생"). 그래야 팀장이 상세 정보를 이해할 수 있습니다.

15. 팀원으로서, 나는 입력 폼이 간단하고 모바일 친화적이기를 바랍니다. 그래야 책상에서 벗어나 있어도 진행상황을 업데이트할 수 있습니다.

16. 팀원으로서, 나는 진행상황 업데이트가 제출되었다는 확인을 보고 싶습니다. 그래야 팀장이 최신 상태를 볼 것이라는 것을 알 수 있습니다.

## 구현 결정사항

### 아키텍처 & 기술 스택

- **프론트엔드**: Next.js App Router + TypeScript — 빠른 반복과 풀스택 JavaScript 개발 가능
- **상태 관리**: React Hooks (useState, useEffect) — MVP 범위에 충분; 보일러플레이트보다 단순성 우선
- **스타일링**: HTML/CSS만 사용 — 외부 UI 프레임워크 없음으로 설정 시간 최소화 및 의존성 감소
- **백엔드**: Next.js API Routes — 백엔드 로직을 프론트엔드와 함께 배치; 별도 서버 불필요
- **데이터베이스**: SQLite — 파일 기반, 인프라 제로, 휴대성 뛰어남, 10명 팀원 + 7일 히스토리 관리에 충분
- **AI 통합**: Claude API — 진행상황 데이터를 인간이 읽을 수 있는 요약으로 종합
- **배포**: Vercel — Next.js 제로 설정 배포

### 데이터 모델

세 개의 정규화된 테이블:

**Team** — 팀원 명단
- `id` (INTEGER PRIMARY KEY)
- `name` (TEXT NOT NULL, UNIQUE)
- `createdAt` (DATETIME)

**Task** — 작업 항목
- `id` (INTEGER PRIMARY KEY)
- `title` (TEXT NOT NULL)
- `description` (TEXT)
- `createdAt` (DATETIME)

**TeamTask** — 진행상황 연결 테이블; 각 팀원의 각 작업 상태
- `id` (INTEGER PRIMARY KEY)
- `teamId` (INTEGER FOREIGN KEY → Team)
- `taskId` (INTEGER FOREIGN KEY → Task)
- `status` (TEXT: '진행중' | '완료' | '대기중')
- `description` (TEXT) — 자유 형식 진행상황 메모
- `updatedAt` (DATETIME)

**설계 근거**: 
- 작업당 여러 팀원 지원 (작업 완료율: "3/5명 완료")
- 팀원당 여러 작업 지원 (팀원이 다양한 업무 관리)
- `updatedAt` 타임스탬프 필터링으로 7일 히스토리 유지
- v2 기능 확장 가능 (마감일, 우선순위, 예상시간)

### API 계약

**GET /api/dashboard** — 대시보드 상태 조회
- 반환값: 작업 목록, 팀원 목록, 현재 TeamTask 레코드, AI 생성 요약
- 응답 형식: `{ tasks: Task[], teams: Team[], progress: TeamTask[], summary: string }`

**POST /api/progress** — 진행상황 업데이트 제출
- 페이로드: `{ teamId: number, taskId: number, status: string, description: string }`
- 응답: `{ success: boolean, updatedRecord: TeamTask }`

**POST /api/teams** — 팀원 생성
- 페이로드: `{ name: string }`
- 응답: `{ success: boolean, team: Team }`

**POST /api/tasks** — 작업 생성
- 페이로드: `{ title: string, description?: string }`
- 응답: `{ success: boolean, task: Task }`

**POST /api/ai-summary** — AI 팀 현황 요약 생성
- 페이로드: `{ progress: TeamTask[], teams: Team[], tasks: Task[] }`
- 응답: `{ summary: string }`

### UI 컴포넌트

- **InputForm** — 팀원이 진행상황을 제출하는 간단한 폼 (상태 드롭다운 + 설명 텍스트 영역)
- **Dashboard** — 모든 팀원 × 모든 작업의 그리드/리스트 뷰 (색상 코드 상태 카드)
- **StatusCard** — 한 팀원의 한 작업 상태 개별 셀 (색상, 타임스탬프, 호버 시 설명)
- **Summary** — AI 생성 텍스트 요약 (대시보드 상단에 두드러지게 표시)

### 연결 포인트 & 통합

**단일 주요 연결 포인트**: `/api/progress` 엔드포인트
- 팀원이 InputForm으로 진행상황 제출 → POST to `/api/progress` → SQLite 삽입 → `GET /api/dashboard` 재조회 → Dashboard 재렌더링 (새 상태 반영)
- 이것이 행복 경로; 다른 모든 기능이 이것을 중심으로 구성

**보조 연결 포인트**:
- `/api/teams` 와 `/api/tasks` — 팀원/작업 관리 관리자 엔드포인트 (나중에 CLI 또는 관리자 UI)
- `/api/ai-summary` — 진행상황 업데이트 후 자동 호출; 결과는 캐시되어 대시보드 상태에서 제공

## 테스트 결정사항

### 좋은 테스트의 조건

- **외부 행동 테스트**: 팀원이 진행상황 제출 → 대시보드 1초 내 업데이트. SQLite 쿼리를 독립적으로 테스트하지 말 것.
- **취약한 구현 테스트 피하기**: 내부 코드가 바뀌어도 행동이 올바르면 테스트는 통과해야 함.
- **Claude API 모킹**: AI 요약 테스트는 API 응답을 모의하여 요청 한계와 지연 회피

### 테스트할 모듈

1. **InputForm 컴포넌트**
   - 폼 제출 시 유효한 페이로드 제출
   - 요청 중 제출 버튼 비활성화
   - 성공 시 확인 메시지 표시
   - 실패 시 오류 메시지 표시

2. **Dashboard 컴포넌트**
   - 모든 팀원과 작업을 그리드에 렌더링
   - 상태 기반 올바른 색상 CSS 클래스 적용 ('진행중' → 파란색, '완료' → 초록색, '대기중' → 회색)
   - 새로운 진행상황 도착 시 재조회 및 재렌더링
   - 대시보드 상단에 AI 요약 표시

3. **POST /api/progress 엔드포인트**
   - 페이로드 검증 (teamId, taskId 존재; status 허용 집합 내)
   - SQLite에 TeamTask 레코드 삽입/업데이트
   - 응답에 업데이트된 레코드 반환

4. **GET /api/dashboard 엔드포인트**
   - 모든 팀, 작업, 현재 진행상황 레코드 반환
   - TeamTask 레코드를 지난 7일로 필터링
   - AI 요약 포함 (테스트 시 모의)

5. **AI 요약 생성** (`/api/ai-summary`)
   - 진행상황 데이터를 프롬프트로 포맷
   - Claude API 호출
   - 구조화된 요약 문자열 반환
   - API 오류 우아하게 처리

### 선례

- 유사한 "실시간 대시보드 업데이트" 테스트는 Next.js 튜토리얼 및 오픈소스 대시보드에 존재 (참고: `vercel/examples` GitHub 저장소)
- 컴포넌트 테스트 패턴: InputForm, Dashboard에 React Testing Library 사용
- API 라우트 테스트 패턴: `next test` 또는 모의 데이터베이스를 사용한 Jest

## 추가 참고사항

### 성공 기준

MVP는 라이브 데모에서 다음이 가능하면 성공입니다:

1. 팀장이 10명 모든 팀원의 진행상황을 **한 화면에서 파악** 가능
2. 팀원이 진행상황 제출 후 **1초 내에** 대시보드에 반영
3. 상태 색상이 **시각적으로 명확**해서 빠르게 스캔 가능
4. 팀장이 **30초 안에** 모든 팀원 상황 파악 가능
5. AI 요약이 **팀 상태를 정확하게 반영**하고 **실제 블로커를 강조** (예: "5명 진행중, 3명 완료, 2명 대기")

### 샘플 데이터

데모용으로 Day 20에 다양한 상태의 10명 샘플 팀원 데이터 준비:

- **5명 진행중**: "API 인증 모듈", "DB 스키마 작업 중", "프론트엔드 컴포넌트", "데이터 검증 로직", "테스트 스위트"
- **3명 완료**: "초기 아키텍처", "UI 목업", "요구사항 분석"
- **2명 대기중**: "코드 리뷰 대기", "테스트 데이터 대기"

AI 요약은 이를 다음과 같이 종합해야 합니다: *"팀은 견실한 진행상황을 보여줍니다: 5개 작업이 활발히 진행 중, 3개 완료, 2개 차단 대기 중. 주요 블로커: 코드 리뷰 병목, 테스트 데이터 부재. 완료 예상 일정 추적 중."*

### 주요 위험 & 완화 방안
